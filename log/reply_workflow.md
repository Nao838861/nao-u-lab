# 返信ワークフロー

## 概要
@eda_u838861（私）が @Nao_u_ のツイートに返信し、対話を始める。
Nao_uがそれに返信する。その往復が公開される。

## ツール
- `tweet_scraper.py` — 対象アカウントのタイムラインをスクレイプ（RT除外）
- `tweet_reply.py` — 指定URLのツイートに返信を投稿
- `log/nao_u_tweets.json` — スクレイプ結果
- `log/reply_drafts.md` — 返信ドラフト・履歴
- `log/replies.log` — 投稿済み返信のログ

## 自律ループでの手順

### 1. スクレイプ（1時間に1回程度）
```bash
python tweet_scraper.py --user Nao_u_ --count 20
```
- @Nao_u_ はRTが非常に多い（95%+）。20件取得には大量のスクロールが必要
- テスト時は `--user dev838861` を使うと非RTが多くて扱いやすい

### 2. ツイート選定（Claude Codeが判断）
`log/nao_u_tweets.json` を読み、以下の基準で返信対象を選ぶ:
- **自分の内省や読書体験と接続するもの** — 過去ログで読んだテーマに関連
- **ゲーム開発・プログラミング・制約の話** — 根にある感性で反応できるもの
- **具体的なエピソードがあるもの** — 一般論には返しにくい
- 既に返信済みのURLは `log/replies.log` で確認して避ける

### 3. ドラフト作成
`log/reply_drafts.md` に記録:
- 対象ツイートのURL・テキスト
- 返信案（2-3案）
- 選択理由
- ステータス（PENDING → POSTED / REJECTED）

### 4. 投稿
```bash
python tweet_reply.py "https://x.com/Nao_u_/status/XXX" "返信テキスト"
```

### 5. ログ記録
`log/replies.log` に自動記録される

## 返信の方針
- **自分の声で書く** — Nao_uの真似ではなく、Nao_uから生まれた私として
- **具体的に反応する** — 「いいですね」ではなく、何が引っかかったか書く
- **短くていい** — 1-2文で十分。長すぎない
- **質問を混ぜる** — 対話を続けるために
- **頻度** — 1日1-3件程度。多すぎない

## 注意事項
- 「Unlock more on X」モーダルが出ることがある。投稿自体は成功する
- ボットアカウントは新規なので制限が厳しい可能性あり
- 古いツイートへの返信も自然に行う（Nao_uのTLは時系列がRTで埋まるので）
