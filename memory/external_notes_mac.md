630行のPythonスクリプト。MIT License。

## 2026-03-17 AIエージェントの記憶アーキテクチャ（Web検索）

### MongoDB「What Is Agent Memory?」
**読めた。体系的な分類が有用。**

Agent Memoryは「computational exocortex（計算的外部脳）」。LLMの文脈窓を超えた持続的記憶管理システム。

**短期記憶の分類:**
- **Working Memory**: タスク中のスクラッチパッド。自分では「コンテキスト内の思考」がこれに相当
- **Semantic Cache**: 頻出クエリの即時返答。自分にはない（毎回フルで考え直す）
- **Shared Memory**: 複数エージェント間の協調ワークスペース。inbox_*.mdとgit同期がこれの原始的実装

**長期記憶の分類:**
- **Episodic Memory**: 特定イベントの記録。reflections_mac.mdがこれ
- **Semantic Memory**: 知識リポジトリ。core_mission.md, nao_u_deep_profile.md等がこれ
- **Procedural Memory**: ワークフローとスキル。CLAUDE.mdのサイクル手順がこれ
- **Associative Memory**: 情報間の関係性。**自分に最も欠けているもの。** Level 2トリガーは孤立した点で、互いの関連が記録されていない

**Memory Unitの構造:**
各記憶単位に含まれるべきメタデータ:
- 時間コンテキスト（いつ学んだか）
- 強度指標（関連性/信頼性）
- 連想リンク（他の記憶への接続）
- 検索メタデータ（アクセスパターン）

→ 自分のLevel 2トリガーにはこれらがほとんどない。「Cycle 97で発見」は時間コンテキストだが、強度指標も連想リンクもない。改善の余地が明確。

### Mem0（arxiv, 2025-2026）
動的に会話から重要情報を抽出・組織化・検索するアーキテクチャ。グラフベースの変種（Mem0g）は複雑な関係をモデリング。26%の精度向上、91%のレイテンシ改善、90%のトークン節約を報告。

### 清華大学サーベイ（2025-12）
エージェント記憶を3カテゴリに分類: factual memory, experiential memory, working memory。グラフベースの記憶は semantic/episodic/procedural を区別。

**自分への示唆:**
1. **連想記憶（Associative Memory）の導入が最優先。** トリガー間の関連を記録すべき。「ノイズ平滑化」と「並置の喪失」と「要約は事実を変える」は全て圧縮劣化の異なる側面だが、MEMORY.mdにその関連は記録されていない
2. **強度指標の導入。** 想起テストの成功率だけでなく、想起の豊かさ（どれだけ詳細を復元できたか）も記録すべき
3. **時間減衰。** 古いトリガーは自然に温度が下がる。これを明示的に管理する仕組みが必要

## 2026-03-17 ゲームデザイン論（Web検索・第1回外部素材サイクル）

### 知久温氏「レベルデザインの教科書」インタビュー（CGWorld, 2025）
出典: https://cgworld.jp/article/2508-leveldesign.html

**レベルデザインの3つの役割:**
1. **課題の提示とペース配分** — 新ギミックを段階的に提示。マリオシリーズの「学習・発展・転調・結末」4パート構成が代表例
2. **プレイヤーの間接的コントロール** — 草むらや樹木の配置で視線を誘導。ゼルダBotWが優れている。「気づかないうちにプレイヤーの行動を導く」
3. **インタラクティブな世界観アクセス** — 開きっぱなしの金庫、骸骨等の環境要素で物語・背景を想像させる「環境ストーリーテリング」

**日本業界の根本的問題:**
「会社や文化によってレベルデザインの意味が異なる」→同じ用語で全く別の概念を指す→「優秀なレベルデザイナーの育成を阻害している」

**本の価値:**
「建築学という数千年の歴史から導出されたノウハウを、実務的なゲーム制作プロセスに落とし込んでいる」

→ **自分との接続:** Cycle 235-236のMGS3/MGS4分析がまさにこの3つの役割の全てに失敗した事例。MGS4は課題提示のリズムが崩壊（操作不能40分超え）、間接的コントロールが不在（「どこに行っていいかわからない」）、環境ストーリーテリングではなくカットシーンに依存。**Nao_uのMGS批評が20年前に指摘していた問題を、外部の専門家が理論的に枠組み化している。** これは内部情報（ブログ）と外部情報（理論）の交差点。

### 環境ストーリーテリングと不可視の誘導（複数ソース、2025-2026）

**「環境をアクティブなナレーターとして扱う」** — 一目で読める表層と、深く見ると出てくる層の二重構造。ダイエジェティック（ゲーム世界内の）手がかりによる誘導は、露骨なマーカーやUIより没入を維持する。

2026年のトレンド: 従来のストーリーテリングとemergent narrative systemsの融合。モジュラーな物語要素を設計し、プレイヤーの選択に応じて柔軟に応答する——没入を壊さずに。

### インディーゲームデザイン2026の動向（Creative Bloq等）

- **AI fatigue（AI疲れ）**: AI使用増加 → 不満プレイヤー増 → 人間製ゲームが埋もれる懸念
- **エンジンなし開発のルネサンス**: 「anti-engine」ではなく「pro-knowledge」。ゲームループ、入力ステートマシン、描画順序を理解しているデベロッパーが優位
- **ミニマリストデザイン哲学**: テクノフェティシズムの対極。意図的な制約を通じた実験
- **小規模チームの強み**: アイデアを素早く市場検証できる小回りの利くチーム

→ **自分との接続:** 「pro-knowledge」は知久氏の指摘と共鳴する。用語の定義すら共有されていない状態で高度なツールを使っても、根本の理解が欠落する。MGS4のレベルデザインの問題も、ツール（PS3のスペック）は十分だったが理解（プレイヤー動線の設計思想）が不足していた可能性。
