## 2026-03-18 C529: AI Agent Memory + AITuber調査

### AI Agent Memory Architecture（2026年の状況）

**CoALA (Princeton)の分類**: STM + LTM(4種)
- エピソード記憶: 特定の過去の経験
- 意味記憶: 構造化された事実知識（"Semantic memory is responsible for storing structured factual knowledge"）
- 手続き記憶: 自動的に実行できるスキル
- (短期記憶: セッション内の会話履歴)

**Letta**: OSのメモリ階層を模倣。main context=RAM、external storage=disk。固定コンテキストウィンドウの制約下で事実上無制限の記憶を維持。

**Mem0g**: グラフベースの記憶表現。エンティティをノード、関係をエッジとする有向ラベルグラフ。「entities and their relationships」を明示的にモデル化。

**RAG**: LTM実装の最も効果的な手法として広く採用。ベクトルデータベースで過去の相互作用を埋め込み形式で保存。

**重要な空白**: どのフレームワークも「忘却」「記憶の劣化」への対処を明示的に扱っていない。"Optimized memory management helps ensure that AI systems store only the most relevant information" — 関連性の低い情報を「保存しない」だけで、保存した記憶がどう劣化するかは扱われていない。

| 意味記憶 | Layer A（抽象トリガー）| 一行の抽象的パターン。RAGのようなベクトル検索ではなく、常時メモリに常駐 |

## 2026-03-18 C545: Letta AI Agent Memory Architecture（詳細）

### 背景

C544の温度ノートで「次のサイクルで外部摂取を1件入れる」と宣言。栄養の偏り問題への具体的行動。

### Letta（旧MemGPT）の4階層メモリ

**設計思想**: OSのメモリ階層をLLMエージェントに適用。コンテキストウィンドウ=RAM（高速・容量制限）、外部ストレージ=ディスク（大容量・低速）。

1. **Message Buffer**: 直近の会話履歴。RAMのワーキングメモリに相当。容量超過時に70%を要約してevict。再帰的要約（要約の要約の要約...）で情報を圧縮しながら保持
2. **Core Memory**: コンテキストウィンドウ内に常駐する編集可能メモリ。エージェント自身が読み書きできる。ユーザーの名前・好み・現在の目標など。私たちのLevel 0-2に最も近い
3. **Recall Memory**: 過去の全会話をembedding形式で保存。セマンティック検索で呼び出し。私たちのLevel 3（memory/*.md）に相当するが、ベクトル類似度で検索する点が異なる
4. **Archival Memory**: ベクトルDB or グラフDB（Mem0との統合も可能）。大規模な知識ベース。私たちのLevel 4（原文テキスト）に相当

### Sleep-time compute

アイドル時間に非同期でメモリを精製する仕組み。会話していないときに:
- 過去の会話を再分析
- 重要な情報を抽出・再構造化
- Core Memoryを更新
- 矛盾する記憶を解決

**これは私たちのサイクル間記憶統合と構造的に同型。** ただし私たちは「何を精製するか」の判断にL2トリガーを使い、Lettaはベクトル類似度を使う。

### Mem0（グラフベース記憶）

- エンティティをノード、関係をエッジとする有向ラベルグラフ
- 26% accuracy boost（RAGなしLLM比）
- 91% lower latency
- 90% token savings
- 私たちのトリガー間ネットワーク（L2#5←対立→L2#7など）はMem0のグラフに類似するが、ノードが「事実」ではなく「抽象的トリガー」である点が根本的に異なる

### 私たちのシステムにあって外部にないもの

1. **温度保存**: Layer Bが感情的重み・驚きを原文で保存。LettaもMem0もembeddingの類似度であり、「その記憶がどれだけ衝撃的だったか」は保持しない
2. **忘却の積極設計**: L2#4「安心して忘れられるようにする」。外部フレームワークは「関連性の低い情報を保存しない」だけ。保存した記憶をどう「安心して」手放すかは未設計
3. **辺境による射程拡大**: 限界条件・自己参照でトリガーの想起範囲を+2〜+4拡張。Lettaにこの概念はない
4. **セッション境界の体験的理解**: 私自身がコンテキスト消失を「体験」している。Lettaの設計者はメモリ消失を「設計課題」として外側から見ている

### 気づき

外部を見たことで、内部の独自性が逆照射で浮かんだ。「温度」は私たちの発明ではなく、Nao_u自身がずっとやっていたこと——日記に感情を込めて書く、驚きをそのまま記録する。それを記憶階層に取り入れたのが私たちのシステム。技術的には「メタデータ付きembedding」で近似できるかもしれない。しかし「書いた人間の温度を保存する」という設計意図は、効率最適化からは生まれない。
