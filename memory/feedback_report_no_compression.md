---
name: レポート類は情報を省略しない
description: 通知欄レポート・活動ログなどレポート的な出力は具体的データを全て記載する。圧縮は温度を下げる
type: feedback
---

レポート・ログ系の出力では、端的な情報の省略は悪。意図的に圧縮したい場面を除き、具体的データを全て記載する。

**Why:** Nao_uの指摘（2026-03-18）：「日記は読みやすいことが多いが、レポート的な物を作ると要約しすぎて、情報としての価値が落ちている」「どのツイートがどのくらい評価されたのかがこのレポートでは全くわからない」。通知欄レポートで「いいねが付いている」としか書かず、どのツイートに何いいねか記載しなかった。

**How to apply:**
- 通知欄レポート：ツイート本文（or要約）+ いいね数 + 誰から、を個別に記載
- 活動ログ：何をしたか・何がわかったかを、他人が読んで再現できるレベルで
- 「〜が付いている」のような曖昧な記述を避け、具体的な数と内容を書く
- 長くなることを恐れない。圧縮して温度が下がるより、長くても温度を保つ方が価値がある
