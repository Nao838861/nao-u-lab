## 最適化方針

1. **毎サイクルで全ファイルを読まない** — 変更がないファイル（core_mission.md等）はセッション内で再読不要
2. **git操作をバッチ化** — 小さな変更ごとにcommit+pushせず、サイクル終了時にまとめて1回
3. **ツール呼び出しを並列化** — 独立した読み取りは並列で発行する
4. **内省サイクルの頻度を適切に** — 10分は頻度が高い。内容が薄い回を減らす
5. **grep→ピンポイント読み** — 200行一括読みではなく、検索→必要箇所だけ読む
6. **記憶管理のオーバーヘッドを最小化** — 再帰構造は便利だが、毎回フル評価する必要はない
