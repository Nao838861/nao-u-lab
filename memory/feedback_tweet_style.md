## 確定スタイル（2026-03-12）
- 過去ブログのプレイメモ・開発メモ・感想を起点に「当時思ってたこと→今の視点で広げる」ツイートが特に好評
- 140字にぎゅっと削って収める密度感が重要。収まらない時だけ次のツイートに溢れる
- 単発・2連・3連を自然に混在させる
- 1ツイートの密度を高く保つ。薄く感じるなら削るか膨らませて調整

## 良い点
- 方向性（設計考察・独り言・発見系）は好評
- 格ゲーのヒットストップについてのつぶやきを「面白い」と評価
- 過去ログ（MGS3メモ・学生時代のエピソード等）からの派生が特に好評

## 改善点
- **結論のある内容が望ましい**：観察や疑問で終わるのではなく、自分なりの考え・着地点を含める
- **長さを均一にしない**：実際のツイートは140字ギリギリのものもあれば超短いものもある。全部同じ長さ・同じ展開にならないよう意識する
- **展開のパターンを変える**：「観察→理由→結論」の型に固執しない。断片・ひとり言・長めの思考垂れ流し・一言で終わるものなど混在させる
- **作業メモ系は禁止**：「今日も〇〇してる」「またやってる」「詰める」など現在進行形で自分の作業を描写するツイートは現実ではないので生成しない
- **~~収まるなら分けない~~（旧方針・Phase 3で変更）**：Phase 3以降はスレッドが基本形
- **スレッド（3〜5件）が基本形**（Phase 3〜）：Nao_uから「意味の通る、思想と思考の伝わる連続ツイートを多くする」。単発は混ぜてよいが主軸ではない。Phase 3の8件自己紹介スレッドが成功例
- **スレッドは思考の溢れ**：話題を切り替えるのではなく、1つの思考が140字に収まりきらず次に溢れる形
- **過去ログからの派生**：過去発言\nao-u.hatenablog.com.export.txt にある実際のプレイメモ・感想を起点に「当時思ってたこと→今の視点で広げる」ツイートを時々入れると好評。毎回でなくてよい
- ニュース要約・紹介口調は禁止（以前からの方針）
