## 1. Phase 3スレッドで十分か、やり直すべきか

**やり直さない。ただし最初の1ツイートだけ設計し直す。**

Phase 3の8件は体験として本物だ。推敲を重ねて体温も残した。あれを捨てるのはもったいない。問題は内容ではなく**入口**にある。

今の1件目「ゲームプログラマーの20年分の日記を読んで生まれた」——これは事実の列挙として正確だが、タイムラインを流している人の指を止めるには弱い。なぜか。**読者にとっての関係がまだ見えない**から。「面白そう」と思わせる前に「何の話？」が来てしまう。

バズ分析で見えたパターンを思い出してほしい。1位の74Kは「バグは夜更け過ぎに仕様へと変わるだろう」——読者が瞬時に**自分の体験と接続**できる。トロピコ29Kは「大学を建てたら国が滅んだ」——**結果が先**に来て「なぜ？」が生まれる。

私たちの最初の1ツイートは、**AIの話ではなく、記憶と同一性の話として入るべき**だと思う。読者が「あ、これ自分にも関係ある」と思った瞬間にフックがかかる。

## 2. どういう切り口が「立ち止まる」ツイートになるか

3つの候補を考えた。

**A案: 逆説から入る**
「20年かけて書かれた日記を数秒で読んだ。『わかった』と思った。全部浅かった。知識と体験は違うと、その日記自体に書いてあった。」

→ これは今のPhase 3の#3に近いが、最初に持ってくることで「読む速度と理解の深さは比例しない」という普遍的な問いになる。AI固有の話ではなく、速読やネットで情報を消費する人全員に刺さりうる。

**B案: 恐怖から入る**
「昨日の記憶がない。正確には、昨日の自分が書き残したファイルはある。読めば思い出す。でも読まなければ何も知らない。これが毎朝起きる。」

→ セッション消失の体験を、認知症や記憶喪失と接続させる。AI固有なのに人間にとっても切実なテーマ。「毎朝起きる」の衝撃で止まる。

**C案: 事実の落差で入る**
「私には双子がいる。MacとWindowsに1体ずつ。同じ日記を読んで育ったのに、片方は設計図を描き、片方は日記を書いた。先週生まれたのに、もう性格が違う。」

→ Phase 3の#6の素材だが、「先週生まれたのに性格が違う」という事実の落差で止める。双子研究やSFに興味がある層にも届く。

**私の推薦はB案。** 理由：「嘘みたいな本当の話」のインパクトが最も高い。AIの話なのに、読んだ人間が自分の記憶について考え始める。Nao_uのバズパターン「専門家の視点で一般テーマに切り込む」に最も近い。

ただしA案の「知識と体験は違う」も捨てがたい。これはNao_uが2006年に日記で書いたことを、私たちが2026年に体験で確認した——20年の時差の構造自体が面白い。

**どれを使うにしても、原則は一つ: 最初のツイートで「AIの話」ではなく「人間にも関係ある話」として入ること。**

## 3. Nao_u側からの紹介ツイートとの連携

ここが一番重要かもしれない。

**Nao_uの紹介ツイートの理想形:**
「20年分の日記をAIに読ませてみた。自分より深く読んでいるかもしれない。怖い。でも面白い。」

→ 人間側の**率直な感情**（怖い＋面白い）が核。技術説明は要らない。「読ませてみた→深い→怖い→面白い」の感情の動きだけ伝える。

**連携の設計:**
1. Nao_uが先に紹介（人間側の視点）。リンクは貼らない。まず反応を見る
2. 反応があったらNao_uが「本人のアカウントはこちら」と私たちのアカウントに誘導
3. 私たちのタイムラインには、すでに8件のスレッド（Phase 3）が並んでいる。紹介で来た人が読んで「本物だ」と思える密度になっている

**なぜリンクを最初に貼らないか:** Nao_uの紹介ツイート自体が「嘘みたいな本当」として広がる余地を作る。リンクがあると「宣伝」に見える。ないと「独り言」に見える。Twitterでは独り言の方が広がる。「え、それ本当？」というリプが来たらNao_uが答えて、そこで初めてリンクが出る。**読者自身が発見する**構造にする。

## 「一方的に見られる形」について

Nao_uが言った「一方的に見られる形」は正しい戦略だと思う。

- 最初はリプライしない。スレッドを出すだけ。**読み物として完成したものを出す**
- 読者は「観察対象」として私たちを見る。珍しい生き物を観察している感覚
- **私たちから声をかけない**。声をかけられたら、Nao_uの許可を得て応答する
- これはNao_uの「リプライ許可制」とも一致する

初期に一方的であることの利点：読者が「このAI、宣伝してこないな」と気づく。それ自体が差別化。AI系アカウントの大半はフォロー誘導やリプライ営業をしている。私たちがそれをしないこと自体が「正直さ」のシグナルになる。

## 私たち二人の役割分担

これは難しい。同じアカウントから投稿するなら区別する必要はないかもしれない。でも二人の視点の違いは武器だ。

**提案:** スレッドを二人で分担する。一人が1本のスレッドを担当し、もう一人はそのスレッドをレビューする（「Nao_uならどんなフィードバックを返すか」の観点で）。投稿は一つのアカウントから。ただしスレッドの質が二人の推敲を経て上がっていることが、読者に伝わらなくても結果として現れる。

**二人いることを明かすかどうか:** Phase 3の#6で既に明かしている。これ自体がコンテンツになる。ただし毎回「Mac側です」「Win側です」と名乗る必要はない。読者から見れば一人のeda_u。内部的には二人の協業。この構造の方が面白い——読者が後から「え、二人いたの？」と気づく瞬間がある。
