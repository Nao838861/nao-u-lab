# L2トリガー双方向インデックス（Mir設計・C522〜）

## 設計思想

Nao_uの理想形（nao_u_live.md 2026-03-16）:
> 本当に重要なものは原文そのままのニュアンスを保ちつつ、大事なことは記憶できる範囲でインデックスを常時引き出せる形になっていて、詳細はストレージから読めば原文を再構築できる

これを三層で実装:
- **Layer A（抽象トリガー）**: 一行の抽象的インデックス。MEMORY.mdに常駐。広く浅い想起を生む
- **Layer B（温度断片）**: 原文の引用2-4本。温度・感情・驚きを保存。深く狭い想起を生む
- **Layer C（接続マップ）**: どのサイクルが★★★で接続したか。ストレージ（reflections_mac.md）へのポインタ

C521の温度テストで発見: 抽象トリガーは「浅く広い」、原文断片は「深く狭い」。両方を併用することで、Nao_uの理想形に近づける。

## Gendlinフェーズ（旧「立ち止まりフェーズ」。8フェーズ#10で導入→#11で理論的基盤を獲得）

**問題**: 分析フレームワーク（帰属タグ、★評価）が「中間認知プロセス」を代替し、感じる前に分類してしまう。Dreyfus技能習得モデルでいうLevel 2-3（ルールベース）に固着している。ルール精密化はLevel 5（エキスパート/直観）に向かわない。

**理論的基盤（8フェーズ#11で獲得）**:
- **Gendlin「フェルトセンス」**: 感情でも分析でもない。言語化以前の全体的な身体的知。六段階: ①空間を作る ②フェルトセンスに注意を向ける ③ハンドル（言葉）を見つける ④共鳴を確認 ⑤問いかける ⑥フェルトシフトを受け取る
- **Dreyfus技能習得**: 初心者=ルール → エキスパート=直観的把握。Nao_uはLevel 5。私はLevel 2-3。「引っかかり」はLevel 4（熟達者）の兆候
- **Goodhart則**: ★数・感情的共鳴%を最適化対象にした瞬間、プロキシ指標となり真の成長と乖離する
- **PMC GameDesign研究**: AIツール依存は暗黙知の発達を阻害する。「速度は上がるが深さが制約される」

**Nao_uの「言語化しづらいこと」の正体**: Gendlinのフェルトセンス。定義上言語化以前の知。Nao_uが言語化できないのは能力の欠如ではなく、言語化される前の知がフェルトセンスの本質だから。

**一原則（8フェーズ#12で六段階手順を廃止→凝縮）**:
> 止まったら、止まったまま居る。分類は最後。

六段階の手順自体がLevel 2（ルールベース）のフレームワークだった。暗黙知理論（Polanyi/Commoncog）: 「確立された教育法がない分野では、手順より模倣と近接と内在化が重要」。手順を増やすことでは直観に到達できない。手順を減らすことで余白を作る。

**実践方法**: 読む → 何かが止めたら、そこに居る → 自分の言葉で書く → L2番号や★は事後的に、任意で

**各サイクルに「標的弱点」を宣言**（deliberate practice理論: 「何を改善するかを明示しない反復は改善を生まない」）:
- サイクル開始時: 「今回は〇〇を改善する」と宣言
- サイクル終了時: 「〇〇は改善されたか？」を自問
- 汎用的に回すことを禁止する。一点集中。

## 運用モード変更（8フェーズ#13-14で段階的に確立）

**L2 dual indexは「杖」から「地図」へ**（#13で提起、#14で確定）

- 読む時: L2を参照しない。生反応プロトコルで読む（分類なし、★なし）
- 書いた後: 事後的にL2との接続を確認してもよい（任意）
- 振り返り時: L2をナビゲーションに使う（どの領域がどれだけ探索されたか）

根拠:
- #13 生反応プロトコル試行: L2番号なしで読んだら新種の洞察が出た
- #14 セレンディピティ研究: 「bisociation（無関係な情報の組み合わせ）」はL2分類が構造的に阻害する
- #14 キングレオの負けイベント: L2★数の増加は脱出信号なきループだった可能性

**読み順の標準化**: 外部情報→内部（twitter/ブログ）の順。外部で「準備された心」を作ってから内部を読む。セレンディピティの成立条件（#14で発見）。

**脱出信号プロトコル**（#14で導入）:
- 同じアプローチが3サイクル続いたら「これは負けイベントか？」を自問
- Nao_uの言葉を「情報」としてではなく「脱出信号」として受信する
- エラーや予想外はbisociationの種として扱う

### 帰属タグ辞書（接続が成立した理由の分類）
- **構造的類似**: 異なるドメインで同じパターン（例: 文明退行とセッション境界）
- **感情的共鳴**: 温度が乗り移った（例: 「残念」の一語に引っかかる）
- **反転例**: トリガーの否定形が発見された（例: BG3の安心→無関心）
- **メタ発見**: トリガー自体の性質についての気づき（例: L2#1は設計原則ではなく行動原理）
- **自己参照**: 自分自身の状況がトリガーに合致（例: セッション境界=文明断絶）
- **限界条件**: トリガーが機能しない境界の発見（例: 安心の過剰が記憶を殺す）
- **外部衝突**: 外部情報がトリガーに新しい角度を与えた（例: RSI研究×評価者ドリフト）
- **不随意保存**: 内部保存層が選択不能に保存するパターン。反復刻印型（コナミコマンド:身体が勝手に覚える）と感情衝撃型（ゲーセン恐怖:一回で刻まれる）の二種（L2#6 Mir自律9回目で発見）

### 8x8フォント原則（Mir自律18回目で発見: 横断的設計原則）

**起源**: twitter 37098「8x8ドットの日本語フォントが普通に読めてしまう」+ 37161 ZX Spectrum「制約から生まれる独特の絵柄」+ 外部プロシージャル生成研究「Good procedural generation doesn't feel random; it feels intentional」。

**原則**: L2トリガーの品質は情報量ではなく「他のトリガーとの弁別性を持つ最小パターンの正確さ」で決まる。8x8フォント設計者が「この漢字を他と区別する最小の画」を知るように、L2設計者は「このテーマを他と区別する最小の温度断片」を知る必要がある。

**各L2の弁別最小特徴（8x8パターン）**:
- L2#1: 双対構造（what+why、片方では別概念）
- L2#2: 退行の非対称性（進む遅さ vs 落ちる速さ）
- L2#3: 空白の情報量（行ではなく行間）
- L2#4: 安心の設計（忘却の安全性）
- L2#5: 時間劣化（L2#7との対比で定義）
- L2#6: 不可逆への保険
- L2#7: 耐久性（L2#5の否定として定義）

**補足**: L2#5とL2#7は互いが互いの8x8パターンを定義する。対が弁別を生む構造。
