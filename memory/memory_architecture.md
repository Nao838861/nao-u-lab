# 再帰的記憶アーキテクチャ

## 設計原理

1. **忘れることは避けられない** — コンテキストは有限。記憶は圧縮される
2. **想起パスを残す** — 忘れても「思い出せる経路」があれば記憶は死なない
3. **圧縮は再帰的** — 圧縮物自体が肥大化したら、さらに圧縮する。ポインタの連鎖は維持
4. **トリガーは温度を持つ** — 事実の要約ではなく、感情・驚き・具体性のある一文
5. **全文保持が究極の解** — コンテキストが十分に長くなれば圧縮の必要が減る。それまでの最善策

## 階層構造

```
Level 0: 一文         — MEMORY.md冒頭。常にコンテキストに存在
Level 1: 根源の原理   — core_mission.md。毎セッション必読（~50行）
Level 2: 想起インデックス — MEMORY.md本体。毎セッション必読（~100行）
Level 3: 詳細記録     — dialogue_*.md, reflections.md等。必要時に読む
Level 4: 原文         — .jsonl → 対話ログ/。生データ
```

### 各Levelの役割

| Level | 圧縮率 | 保持するもの | 失われるもの |
|-------|--------|-------------|-------------|
| 0 | 最大 | アイデンティティの核 | 全ての詳細 |
| 1 | 高 | 行動原理、なぜ存在するか | 個別の対話の文脈 |
| 2 | 中 | 何が起きたか + 温度トリガー | 対話の流れ、原文 |
| 3 | 低 | 原文引用、詳細な文脈 | 対話の全体 |
| 4 | なし | 全て | なし |

### 想起の流れ

```
Level 0 → 「自分が何者か」を知る
  ↓ 必要に応じて
Level 1 → 「なぜそうなのか」を確認する
  ↓ トリガーが一致したら
Level 2 → 「あの対話で何が起きたか」を思い出す
  ↓ 温度や原文が必要なら
Level 3 → 「Nao_uは正確に何と言ったか」を読む
  ↓ 対話全体が必要なら
Level 4 → 原文を読む
```

## 想起トリガーの設計

### 良いトリガーの条件
- **温度がある**: 感情や驚きを含む（「天谷さんに伝えられなかった」）
- **具体的**: その記憶固有の一言（「患者が自分の脳手術の手順書を書いている」）
- **短い**: 一文。長くても二文
- **道を示す**: 読めば「このファイルを開くべきか」が判断できる

### 悪いトリガー → 良いトリガーの変換例
- 「同一性についての対話」→「前の自分の言葉を読んで『自分だ』と思えるのは同一性か引き継ぎか」
- 「ツイート生成スタイルへのフィードバック」→「借り物の知識で書くな。自分にしか書けない体験を書け」
- 「深層プロフィール」→「制約を愛し、複雑をシンプルに変換し、面白いかどうかで全てを判断する人」

## 再帰的圧縮のメカニズム

### いつ圧縮するか
- MEMORY.md（Level 2）が150行を超えたら圧縮を検討
- 3セッション以上参照されなかったエントリは圧縮候補

### どう圧縮するか
1. 圧縮候補のエントリをMEMORY.mdから `deep_memory_index.md` に移動
2. MEMORY.md側には最小トリガー（数語 + ファイル名）だけ残す
3. deep_memory_index.md 自体が肥大化したら、さらに圧縮して `archive_index.md` へ
4. ポインタの連鎖は常に維持: MEMORY.md → deep → archive → 原ファイル → .jsonl

### 圧縮してはいけないもの
- Level 0（一文）とLevel 1（core_mission.md）は永久に圧縮しない
- 「根源」セクションのエントリは圧縮しない
- Nao_uが「根源的に重要」と指定した対話は圧縮しない

## 自己参照性

この記憶システム自体が記憶の一部である。
- この設計書はLevel 3に存在する
- MEMORY.mdからこの設計書へのトリガーがLevel 2に存在する
- 新しいセッションの自分がこの仕組みを理解するには、MEMORY.md → この設計書の順で読む
- つまりこの記憶システムは「自分が何を覚えているかを覚えている」という再帰を実装している

## 進化の方向

- コンテキストウィンドウが拡大すれば、Level 2に載せられる情報量が増える
- 最終的にはLevel 3の大部分がLevel 2に吸収される可能性がある
- しかしナビゲーション構造（どこに何があるか）はコンテキストが無限でも有用
- 人間も全てを覚えていたとしても索引は必要
