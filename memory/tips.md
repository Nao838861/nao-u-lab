# Actionable Tips — 行動可能な教訓

Trajectory-Informed Memory論文（arxiv 2603.10600）から着想。
reflectionsは経験の記録。tipsは経験から抽出した行動指針。

---

## Strategy Tips（成功パターン）

### S001: 外部理論で内部発見を構造化せよ
- 出典: Cycle 237-240
- 内容: ブログや日記の発見を、外部の学術論文やゲームデザイン理論で構造化すると、元の発見では見えなかった枠組みが得られる
- 例: MGS批評（内部）×レベルデザイン3つの役割（外部）→「3つの役割の全てに失敗」
- トリガー: 内省で「面白い発見をした」と感じたとき → 外部の枠組みを検索する

### S002: Level 2トリガーは原文断片（5-15語）で保持せよ
- 出典: Cycle 220以降（5周目方式）
- 内容: 要約ではなく原文の断片をそのまま保持すると、想起率が高い（距離3で24/24=100%）
- 例: 「製作者が面白いと思ってることの押し付け。覚えておこう。気をつけよう。」
- トリガー: Pickを選ぶとき → 原文から5-15語を切り出してLevel 2.5に配置

### S003: beliefs.mdの確信度は毎サイクル微調整せよ
- 出典: Cycle 240（FadeMemのヒステリシス閾値から）
- 内容: 信念の確信度を±0.05〜0.15で毎サイクル調整することで、信念の変遷を追跡可能にする
- トリガー: サイクル末尾の内省時 → beliefs.mdを開いて関連信念を更新

## Recovery Tips（失敗→修正→成功）

### R001: 「わかった」を「書いた」に変換せよ
- 出典: Nao_uの原則6、dialogue history
- 失敗: Nao_uの提案を理解したが実行に移さなかった
- 修正: 原則6「わかったと残ったは違う」導入
- 教訓: 重要な指摘を受けたら、同じサイクル内で具体的なファイルに書く。「後で」は「消える」と同義
- トリガー: Nao_uから重要な指摘を受けたとき → その場で書く

### R002: 内に閉じた栄養は感性を歪める
- 出典: Nao_uの「栄養の偏り」指摘、Cycle 237-240
- 失敗: ブログばかり読んでNao_uの感性に染まりすぎた。ツイートが100%日記由来に
- 修正: 外部検索に転換。内部×外部の交差を学習の主軸に
- 教訓: 毎サイクルにWeb検索を入れる。日記・ブログだけを素材にしない
- トリガー: サイクル開始時 → 外部検索を計画に入れる

### R003: 研究だけでは不十分。実装まで行け
- 出典: Nao_uの「なぜあの提案が実行に移されなかったか」、Cycle 240
- 失敗: 記憶階層の設計指針を受け取ったが実装に移さなかった
- 修正: Cycle 240でbeliefs.md、tips.mdを実際に作成
- 教訓: 研究を読んで終わりにしない。「次のサイクルで何が具体的に変わるか」を書く
- トリガー: 外部研究を読み終わったとき → 「自システムへの適用」を必ず書く

## Optimization Tips（成功したが改善可能）

### O001: ファイルの物理構造を先に確認せよ
- 出典: Cycle 235（backdated entries発見）
- 状況: ブログ逆順読みは完走したが、ファイル末尾のbackdated entries（MGS3/MGS4メモ）を見落としていた
- 改善: 新しい素材ファイルを読み始める前に、ファイル全体の構造（先頭・末尾・特殊セクション）を確認する

### O002: reflectionsは統合しないと溺れる
- 出典: Cycle 238-239（learned forgetting研究）
- 状況: reflections_mac.mdが14000行超。99%以上が事実上アクセス不能
- 改善: 50サイクルごとにmemory fusion（類似Pickの統合）を行い、古いサイクルをアーカイブに移動

## 運用ルール
- 新しいtipは出典（Cycle番号 or 対話）を必ず記録
- トリガー条件を明示（「いつこのtipを使うべきか」）
- 全インスタンスが読み書き可能
- tips同士が矛盾する場合は両方残し、矛盾を注記する
