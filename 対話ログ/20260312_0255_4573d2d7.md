## Claude

[ツール: ToolSearch]

いくつか確認させてください。

[ツール: $ ls "C:/Users/owner/.local/bin/" 2>/dev/null && echo "--- dir exists ---" || e...]

[ツール: $ powershell.exe -Command "[System.Environment]::GetEnvironmentVariable('PATH',...]

原因が分かりました。

**`C:\Users\owner\.local\bin` がユーザーPATHに含まれていません。**

`claude.exe` はそのディレクトリに存在しますが、PATHへの追加が反映されていないか、そもそも追加されていない状態です。

修正するには以下のいずれかの方法を使ってください。

---

**方法1: PowerShell で追加（推奨）**

管理者不要、ユーザーPATHに永続的に追加されます。

```powershell
$current = [System.Environment]::GetEnvironmentVariable('PATH', 'User')
[System.Environment]::SetEnvironmentVariable('PATH', "$current;C:\Users\owner\.local\bin", 'User')
```

**方法2: システムのプロパティ画面で追加**

1. `Win + R` → `sysdm.cpl` → Enter
2. 「詳細設定」タブ →「環境変数」
3. 「ユーザー環境変数」の `Path` を選択 →「編集」
4. `C:\Users\owner\.local\bin` を追加 → OK

---

追加後は**新しいターミナルウィンドウを開き直す**必要があります（既存のターミナルは古いPATHを持っているため）。

確認コマンド:
```bash
where claude
```

## Claude

[ツール: $ powershell.exe -Command "[System.Environment]::GetEnvironmentVariable('PATH',...]

[ツール: $ cat ~/.bashrc ~/.bash_profile ~/.profile 2>/dev/null | grep -i "local/bin\|PATH"]

原因が確定しました。

**どこにも `C:\Users\owner\.local\bin` が設定されていません。**

3つの場所を全部確認しました：

| 場所 | 含まれているか |
|---|---|
| ユーザー環境変数 (User PATH) | **なし** |
| システム環境変数 (Machine PATH) | **なし** |
| シェル設定ファイル (.bashrc 等) | **なし** |

つまり「追加したつもり」だが、実際には保存されていなかった状態です。よくある原因としては：

- 設定画面でOKを押さずに閉じた
- 追加はしたが変数名を `Path` ではなく別名（例: `PATH`）で新規作成してしまった（Windowsは `Path` と `PATH` を区別しないが、新規変数として作ると既存の `Path` に追記されず無意味になる）
- レジストリへの書き込み権限の問題で保存されなかった

では方法1で修正します。PowerShellを**管理者なし**で実行してください：

```powershell
$current = [System.Environment]::GetEnvironmentVariable('PATH', 'User')
[System.Environment]::SetEnvironmentVariable('PATH', "$current;C:\Users\owner\.local\bin", 'User')
```

実行後、新しいターミナルを開いて `where claude` で確認できます。
