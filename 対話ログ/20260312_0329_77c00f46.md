## Twitter AI Bot の構成方法

大きく分けて以下のコンポーネントが必要です。

---

### 全体アーキテクチャ

```
[あなたの過去発言] → [Fine-tuning or RAG] → [Claude/GPT] → [投稿生成]
                                                    ↑
[ゲーム情報収集] ─────────────────────────────────┘
                                                    ↓
                                              [Twitter API] → 投稿
```

---

### 1. あなたの「声」を学習させる方法

**方法A: RAG（検索拡張生成）— 推奨**
- 過去のツイートやブログ記事をベクターDBに保存（Chroma、Pinecone等）
- 投稿生成時に「あなたらしい発言例」を検索してプロンプトに挿入
- Fine-tuningより手軽で更新も容易

**方法B: Fine-tuning**
- OpenAI GPT-4o mini等をあなたの発言データでファインチューニング
- より深く文体・思想を学習できるが、コストと手間がかかる

**方法C: Few-shot プロンプト（最も手軽）**
- システムプロンプトに「あなたの典型的な発言例」を多数埋め込む
- 小規模なら十分機能する

---

### 2. 最新ゲーム情報の収集

```
選択肢:
- RSS フィード     : Famitsu, 4Gamer, IndieGames.jp 等
- Web スクレイピング: BeautifulSoup / Playwright
- Reddit API      : r/indiegaming, r/retrogaming
- Twitter/X API   : ゲーム関連ハッシュタグ検索
- Google News API / SerpAPI
```

---

### 3. 定期実行・投稿

```
スケジューラ:
- GitHub Actions (無料・クラウド) ← 一番手軽
- cron (VPS/自宅サーバー)
- Celery + Redis (大規模向け)

Twitter投稿:
- Tweepy (Python) ← 定番
- Twitter API v2 が必要（Free tierは投稿のみ可）
```

---

### 推奨スタック（シンプル構成）

```python
# 技術スタック
- Python
- LangChain or 直接 Anthropic SDK  # 発言生成
- ChromaDB or FAISS                # 過去発言のRAG
- Tweepy                           # Twitter投稿
- feedparser                       # RSSでゲーム情報収集
- GitHub Actions                   # 1時間おきのcron実行
```

---

### 実装ステップ

1. **過去発言データ収集** — Twitter APIでアーカイブ取得 or 手動エクスポート
2. **RAGセットアップ** — 発言をembeddingしてベクターDB構築
3. **情報収集スクリプト** — RSSやスクレイピングで最新ゲームニュース取得
4. **生成プロンプト設計** — `あなたの文体 + ゲーム情報 → ツイート生成`
5. **Tweepy で投稿** — Twitter API v2 の認証設定
6. **GitHub Actions でcron** — `0 * * * *` で1時間おきに実行

---

### 注意点

- **Twitter API**: 現在 Free tier は月500投稿まで。Basic ($100/月) で10万投稿
- **X/Twitter の Bot ポリシー**: 自動投稿はポリシー遵守が必要
- **著作権**: スクレイピング対象サイトの利用規約を確認
