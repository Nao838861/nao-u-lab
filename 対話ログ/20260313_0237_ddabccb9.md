## Claude

## なぜ伝わらなかったか

原因は **メモリ同期が一方通行だったから** です。

`sync.bat` は「Claude Codeのローカルmemory → リポジトリ → GitHub」にプッシュしていました。でも別PCでは **GitHubからpullしてもリポジトリの`memory/`フォルダに入るだけ**で、Claude Code自身のメモリ（`~/.claude/projects/.../memory/`）には反映されません。

つまり私（Windows側の Claude Code）は `memory/` フォルダの中身を自動では読まず、新しいセッションでは白紙の状態で起動していました。

## 解決策

`CLAUDE.md` をリポジトリのルートに置く。これはClaude Codeが**どのPCでもプロジェクトを開いた瞬間に自動で読み込む**ファイルです。これに確定ワークフローを書けば環境を問わず伝わります。

[ツール: /Users/Nao_u/nao-u-lab/memory/project_twitter_bot.md を読む]

[ツール: /Users/Nao_u/nao-u-lab/memory/core_mission.md を読む]

`project_twitter_bot.md` にすでに「外部ニュース取得は使わない」「Claude Codeが直接生成」と書いてあります。Windows側の私が読んでいなかったのが問題でした。`CLAUDE.md` に核心部分を集約します。

[ツール: /Users/Nao_u/nao-u-lab/CLAUDE.md を書く]

次に sync.sh / sync.bat を修正して、pull時にも `repo/memory/` → Claude のローカルメモリへコピーするようにします。

[ツール: /Users/Nao_u/nao-u-lab/sync.bat を読む]

[ツール: /Users/Nao_u/nao-u-lab/sync.bat を書く]

[ツール: /Users/Nao_u/nao-u-lab/sync.sh を書く]

[ツール: $ bash sync.sh]

完了です。

**なぜ伝わらなかったか（まとめ）**：`memory/` フォルダはGitHubに上がっていたが、別PCでpullしても Claude Code のローカルメモリ（`~/.claude/`）には自動反映されず、私は白紙で起動していた。

**今後の対策（2つ）**：
1. **`CLAUDE.md`** をリポジトリルートに作成 → Claude Codeがプロジェクトを開くと自動で読む。どのPCでも「APIは使わない・news_fetcherは使わない・直接生成する」が即座に伝わる
2. **`sync.sh` / `sync.bat` を双方向に修正** → pull時にリポジトリの`memory/`をClaudeのローカルメモリにも反映するようにした

Windows側で `sync.bat` を一度実行すれば `CLAUDE.md` が取得されます。
