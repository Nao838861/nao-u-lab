## Claude Code のレート制限確認方法

質問いただいた4つのポイントについて、まとめた内容は以下の通りです。

### 1. CLI 上で残量が表示される場所

CLI 上では **直接的なレート制限残量の表示はありません** が、以下の方法で使用状況を確認できます：

- **`/cost` コマンド**: API トークン使用量を確認できます（Pro・Max加入者は `/stats` を使用して使用パターンを確認）
- **`/stats` コマンド**: Claude Pro/Max ユーザー向けに使用パターンを表示
- **ステータスラインの設定**: 常時表示させることで、コンテキストウィンドウ使用量を監視可能

### 2. リミットに達した時の挙動

リミットに達した場合：

- **API Error: Rate limit reached** メッセージが表示されます
- Fast mode を使用している場合は、自動的に標準 Opus 4.6 にフォールバックします
- 実際には 3 つの独立したレート制限システムがあるため、ダッシュボード上で使用率が低い（6%）場合でも、スループット制限に達することがあります

### 3. Web ダッシュボードでの確認方法

**Claude Console** で確認できます：

- **Limits ページ**: 現在のレート制限を確認
- **Usage ページ**: 詳細な使用状況を視覚化
  - トークンとリクエストのチャート
  - 入出力トークンの分別表示
  - キャッシュ率の表示
  - RPM（1分あたりのリクエスト数）、ITPM（1分あたりの入力トークン）、OTPM（1分あたりの出力トークン）の監視チャート

**アクセス**: https://platform.claude.com/settings/limits

### 4. Pro と Pro Max での制限の違い

Claude Code は **API トークン消費量に基づいて課金** されるため、Pro・Max 加入者は以下の特徴があります：

- **加入者**: サブスクリプションに使用量が含まれており、`/cost` データは課金目的ではなく参考情報です
- **API ユーザー**: `/cost` コマンドで詳細なトークン使用量を確認可能
- **チーム使用時**: Claude Console で ワークスペース単位のレート制限を設定可能
